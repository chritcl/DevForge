# ADR-0002：使用 SQLite 作为业务数据事实来源

## 状态

Accepted

## 背景

DevForge 需要持久化工作区、文档、符号、对话、任务、配置等业务数据。需要选择数据库方案。

## 决策

使用 SQLite（WAL 模式）作为所有业务元数据的唯一事实来源。Tantivy 和向量索引属于可重建数据，不作为权威存储。

## 备选方案

- PostgreSQL
- SQLite
- 自定义二进制格式
- 纯文件系统存储

## 原因

- 零配置，桌面应用无需额外安装数据库服务
- 单文件便于备份和迁移
- WAL 模式支持并发读写
- SQLx 提供编译期 SQL 检查
- 足以支撑数十个工作区和中大型本地代码仓库

## 后果

- 第一版使用单全局数据库，后续可评估是否按工作区拆分
- 大型正文和索引文件保存到文件系统，不全部塞进数据库
- 需要维护数据库迁移脚本
- 索引损坏时可从 SQLite 重建

## 以后何时重新评估

- 当单个工作区数据量导致 SQLite 性能瓶颈时
- 当需要支持多设备实时同步时
- 当需要团队共享数据库时
- 当 SQLite WAL 模式在特定平台上出现稳定性问题时
