# ADR-0005：采用人工审批的受控 Agent 模型

## 状态

Accepted

## 背景

需要让 AI 能够辅助代码修改，但必须保证安全性和可回滚性。

## 决策

AI 只负责提出意图（结构化 Tool Call），Rust 后台负责实际执行。所有副作用操作必须经过 Policy Engine 评估和人工批准。AI 不能批准自己提出的操作。

## 备选方案

- 完全自主 Agent（AI 直接执行所有操作）
- 人工审批 Agent（推荐）
- 纯只读 AI（不允许任何修改）
- 沙箱隔离 Agent

## 原因

- 批准对象必须是确定的文件、命令和参数，不能批准模糊意图
- 批准后如果操作内容发生变化，原批准立即失效
- 所有副作用操作必须进入任务时间线和审计日志
- 用户随时可以取消任务

## 后果

- Agent 实现分四个子阶段：Plan 模式 → Patch 生成 → 隔离应用 → 命令执行
- 需要实现 Change Session、Approval Hash、PathGuard 等安全机制
- 第一版使用 Git Worktree 隔离 Agent 修改
- "在临时目录运行"不等于真正的安全沙箱，UI 不能误导用户
- 命令执行属于"受控执行与进程隔离"，不是强安全沙箱

## 以后何时重新评估

- 当用户对审批流程的效率反馈持续负面时
- 当需要支持团队协作场景，多人审批流程变得复杂时
- 当容器化或沙箱技术成熟到可以提供真正的安全隔离时
- 当 AI 模型的可靠性显著提高，可以考虑扩大自动执行范围时
