# 工作区导入实施计划

## 目标

用户可以创建开发工作区，添加本地 Git 仓库和文档目录，在 DevForge 中浏览代码。

## 迭代 2：本地工作区

### 工作内容

#### 2.1 工作区管理

- 实现 CreateWorkspace Tauri Command
- 实现工作区列表查询
- 实现工作区编辑（名称、描述）
- 实现工作区归档
- 实现最近工作区记录
- React 工作区创建页面
- React 工作区列表页面

#### 2.2 数据源添加

- 实现 AddSource Tauri Command（本地 Git 仓库）
- 实现 AddSource Tauri Command（普通文档目录）
- 实现 RemoveSource Tauri Command
- 实现数据源列表查询
- React 数据源添加对话框
- React 数据源列表

#### 2.3 文件发现

- 实现目录扫描（递归遍历）
- 实现 .gitignore 解析
- 实现 .devforgeignore 解析
- 实现文件类型识别
- 实现二进制文件检测
- 实现文件大小限制
- 实现敏感文件识别（.env、*.pem、*.key 等）
- 将发现的文件写入 documents 表

#### 2.4 文件树

- 实现文件树查询 Tauri Command（支持分页和懒加载）
- React 工作区资源管理器组件
- 目录懒加载（点击展开）
- 文件类型图标
- 文件状态标记（M/A/D/I/V/!）

#### 2.5 文件查看器

- Monaco Editor 只读查看器
- Markdown 富文本查看器
- 文件基础预览（图片、PDF 等）
- 标签页管理（打开、关闭、切换）

#### 2.6 布局持久化

- 保存工作区打开标签
- 保存活动标签
- 保存面板大小
- 应用重启后恢复

### 交付验证

```text
创建工作区
→ 添加三个本地仓库
→ 浏览文件树
→ 打开源码文件
→ 打开 Markdown 文件
→ 关闭应用
→ 重新打开，标签和布局恢复
```

### 验收标准

- 可导入至少三个本地仓库到同一工作区
- 大型目录不会一次全部传给 React（懒加载）
- 敏感文件默认不读取正文
- 文件树支持增量刷新
- 应用重启后可以恢复工作区和打开标签
- 删除工作区不会删除原始仓库
- 文件路径逃逸测试全部通过
